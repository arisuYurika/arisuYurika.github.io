    CSS样式笔记

    御坂10043 2020/5/29 CSS

    # 一、盒子模型

    盒子包含内容：内容（content）、内边距（padding）、边框（border）、外边距（margin）。

    # 1. 边框（border）

    # 1.1 边框总属性与简写

    语法（没有顺序指定）：border：border-width || border-style || border-color

    例子：

    border: 1px solid red;
/* 或者 */
border-top: 1px solid red;
    1
    2
    3
    边框总属性 作用
    border-width 边框粗细
    border-style 边框样式（虚线、实线等）
    border-color 边框颜色

    # 1.2 边框样式（border-style）

    border-style 样式值 作用
    border-style：none 无样式
    border-style：solid 单实线
    border-style：dashed 虚线
    border-style：dotted 点线

    # 1.3 边框粗细（border-width）与边框颜色（border-color）

    border-width:18px;
border-color:red;
    1
    2

    # 2. 内边距（padding）

    # 2.1 内边距总属性与简写

    内边距没有样式设置和颜色设置，能设置 top/bottom/left/right。

    它的简写可以有一到四个属性值：

    值的个数 表达的含义
    padding：1px 上下左右都是 1px 的内边距
    padding：1px 2px 上下内边距是 1px，左右是 2px
    padding：1px 2px 3px 上内边距是 1px，左右内边距是 2px，下内边距是 3px
    padding：1px 2px 3px 4px 上 1px，右 2px，下 3px，左 4px （顺时针）

    # 2.2 内边距的实际应用

    # 2.2.1 运用内边距撑大盒子的特性

    每个盒子里的内容、字数都不一样，设置内边距使其撑开盒子，而不直接设置宽度。

    有时，padding 撑大盒子的特性也用于修改宽度。

    # 3. 外边距（margin）

    # 3.1 外边距总属性与简写

    外边距没有样式设置和颜色设置，能设置 top/bottom/left/right。

    它的简写可以有一到四个属性值：

    值的个数 表达的含义
    margin：1px 上下左右都是 1px 的外边距
    margin：1px 2px 上下外边距是 1px，左右是 2px
    margin：1px 2px 3px 上外边距是 1px，左右外边距是 2px，下外边距是 3px
    margin：1px 2px 3px 4px 上 1px，右 2px，下 3px，左 4px （顺时针）
    • 外边距的相关设置与内边距完全一致

    # 3.2 外边距的实际应用

    # 3.2.1 外边距使盒子水平居中（居中问题）

    外边距的设置可以使**块级盒子**水平居中，需要两个条件

    • 盒子指定了宽度（width）
    • 盒子左右的外边距（padding）都设置为 auto
    .header {
  width: 960px;
  margin: 0 auto;
}

    2
    3
    4

    以下两种写法也可以：

    margin: auto;
/*============*/
margin-left: auto;
margin-right: auto;
    1
    2
    3

    行内元素与行内块元素水平居中的方法：给父元素添加 text-align：center 即可。

    # 3.2.2 盒子之间的外边距合并（塌陷问题）

    使用外边距（margin）定义元素的垂直外边距时，可能会出现外边距的合并。

    1. 相邻块元素垂直外边距的合并。

      上下相邻的两块元素相遇时，上方元素的 margin-bottom 与下方元素的 margin-top 不会相加，而是取两个值当中较大者。

    解决方法：只给一个盒子添加 margin 值。

    1. 镶嵌块元素垂直外边距的塌陷。

      父元素有上外边距的同时子元素也有上外边距，此时父元素会塌陷较大的外边距值。

    解决方法：

    • 可以为父元素定义上边框。
    • 可以为父元素定义上内边距。
    • 可以为父元素添加 overflow：hidden。

    浮动、固定、绝对定位的盒子不会有塌陷问题

    # 4. 盒子的应用要点

    # 4.1 盒子的整体大小管理

    当指定了盒子的宽与高（width 与 height）时，设置的边框、内边距会撑开（撑大）盒子。

    如果想要盒子与效果图大小一致，应使宽高（width 与 height）减去设置好了的边框与内边距等数值。

    行内元素为了照顾兼容性，尽量只设置左右内外边距，不要设置上下内外边距。 块级元素和行内块元素没有这个顾虑。

    # 4.2 网页元素默认的内外边距清除

    元素自带默认的内外边距，不同浏览器默认的元素内外边距也不一致，在布局前，需要清除元素自带的内外边距。

    * {
  padding: 0; /* 清除内边距 */
  margin: 0; /* 清除外边距 */
}

    # 5. css3 的盒子功能

    # 5.1 圆角边框

    border-radius 属性 用于设置元素的外边框圆角，该属性可以设置四个值，分别代表左上角，右上角，右下角，左下角。（顺时针）

    简写语法： border-radius: length;

    圆角属性设置 含义
    border-top-left-radius 左上
    border-top-right-radius 右上
    border-bottom-left-radius 左下
    border-bottom-right-radius 右下

    # 5.2 盒子的阴影

    box-shadow 属性 用于设置盒子的阴影，该属性可以设置四个值，分别代表左上角，右上角，右下角，左下角。（顺时针）

    语法：box-shadow: h-shadow v-shadow blur spread color inset

    属性的值 含义
    h-shadow 必选。水平阴影的位置，可以负值
    v-shadow 必选。垂直阴影的位置，可以负值
    blur 可选。模糊距离
    spread 可选。阴影的尺寸
    color 可选。阴影的颜色
    inset 可选。将外部阴影（outset）改为内部阴影（默认为外部阴影，但是不能单独设置）
    • 盒子阴影不占空间，不会影响其他盒子

    # 二、浮动（float）

    # 1. 浮动的定义

    浮动的典型运用：让多个 块级元素 一行内排列显示，纵向块级找标准流，横向块级找浮动。行内块元素可以实现块级元素一行显示，但是它们之间会有较大的空白缝隙，很难控制。

    定义：float 属性用于创建浮动框，将其移动到一边，直到左边缘或者右边缘触及到包含其的块或另一个浮动框的边缘。

    语法： { float: 属性值; }

    属性值 含义
    float:none 元素不浮动（默认）
    float:left 元素左浮动
    float:right 元素右浮动

    # 2. 浮动的特性

    1. 脱标
    2. 一行显示且顶部对齐
    3. 具有行内块元素的特性
    4. 贴在一起的浮动元素没有缝隙，如果超出父级盒子的宽度，则会另起一行对齐
    5. 浮动盒子的大小由内容来决定
    6. 但是父级盒子不会被子盒子撑开（需要清除浮动）
    • 标准父元素排列上下位置，内部子元素采取浮动排列左右位置。
    • 子盒子的兄弟盒子也应该浮动，以免发生问题。
    • 浮动的盒子只会影响盒子后面的标准流，不会影响前面的标准流。

    # 3. 浮动的应用场景

    • 标准流的父元素排列上下位置，之后内部子元素采取浮动排列左右位置。

    # 4. 清除浮动（浮动盒子与父盒子的高度设置问题）

    理想状态：子盒子撑开父盒子的高度

    如果父元素直接设置高度，则不存在问题，但是往往父盒子不方便直接设置高度，内部的浮动子盒子又不占位置，无法撑开父盒子。

    清除浮动元素造成的影响，父级就会根据浮动的子盒子自动检测高度。

    语法：{clear: 属性值;}

    属性值 含义
    clear:left 不允许该元素左侧有浮动元素（清除左侧浮动的影响）
    clear:right 不允许该元素右侧有浮动元素（清除右侧浮动的影响）
    clear:both （重点） 同时清除左右两侧浮动的影响

    清除方法总结：

    清除浮动的方式 优点 缺点
    额外标签法（隔墙法） 通俗易懂，书写方便 添加许多无意义的标签，结构化较差
    父级元素 overflow：hidden 书写简单 溢出隐藏
    父级元素 after 伪元素 结构语义化正确 IE6-7 不支持
    父级元素双伪元素 结构语义化正确 IE6-7 不支持
