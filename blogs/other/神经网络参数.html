    self 在 Python 中是指类的实例。它是当前正在操作的对象的引用，允许您访问该对象的属性和方法。

    self.DATASET：使用的数据集

    POLARITIES_DIM：极性维度，即情感分类的类别数目，对于情感分析通常设置为3（正向、中性、负向）。

    EMBEDDING_DIM：词向量嵌入的维度，用于表示每个单词在向量空间中的位置。

    LEARNING_RATE：学习率，表示在优化过程中调整权重的速度。

    INITIALIZER：初始化器，定义了各层权重的初始值，这里使用了均匀分布随机初始化器。

    REGULARIZER：正则化器，用于防止过拟合，这里使用L2正则化器。

    LSTM_PARAMS：LSTM层的参数配置，包括LSTM单元数、激活函数、权重初始化、正则化等。

    这些参数是用于定义一个LSTM模型的参数。下面是每个参数的解释： units: LSTM中的神经元数。 activation: 激活函数，用于计算输出序列。 recurrent_activation: 递归激活函数，用于计算递归状态。 kernel_initializer: 核Tensor的初始化程序。 recurrent_initializer: 递归核Tensor的初始化程序。 bias_initializer: 偏差向量的初始化程序。 kernel_regularizer: 核Tensor的正则化程序。 recurrent_regularizer: 递归核Tensor的正则化程序。 bias_regularizer: 偏差向量的正则化程序。 dropout: 在0到1之间的浮点数，表示输入单元的dropout比率。 recurrent_dropout: 在0到1之间的浮点数，表示递归单元的dropout比率。 这些参数可以用于控制LSTM的性能和行为，例如增加或减少神经元数量、使用不同的激活函数、添加正则化等。

    MAX_SEQUENCE_LENGTH：序列长度的限制，超过该长度的序列将被截断。

    MAX_ASPECT_LENGTH：方面的最大长度，用于提取句子中的方面信息。

    BATCH_SIZE：每次迭代训练时所用的样本数。

    EPOCHS：要训练的总轮数。

    def precision(y_true, y_pred):
  true_positives = K.sum(K.round(K.clip(y_true * y_pred, 0, 1)))
  predicted_positives = K.sum(K.round(K.clip(y_pred, 0, 1)))
  return true_positives / (predicted_positives + K.epsilon())

    这段代码实现了一个精度（precision）指标的计算函数，其中 y_true 表示真实标签，y_pred 表示模型预测的标签。该函数使用了Keras后端的函数和操作。

    具体解释如下：

    true_positives = K.sum(K.round(K.clip(y_true * y_pred, 0, 1)))：计算真正例数，即在真实标签和预测标签都为1的情况下，正确预测的样本数。首先将相乘结果截断到 [0,1] 区间内，然后四舍五入取整，最后计算总和。

    predicted_positives = K.sum(K.round(K.clip(y_pred, 0, 1)))：计算预测正例数，即所有被预测为正例的样本数。同样先将预测标签值截断到 [0,1] 区间，取整并计算总和。

    return true_positives / (predicted_positives + K.epsilon())：最终返回精度值，即真正例数除以预测正例数加上一个极小值 epsilon，用于稳定计算。

    需要注意的是，这里使用了Keras中的一些通用函数和操作，其中 K 是Keras的后端（backend）引擎，可以是TensorFlow、Theano等。epsilon() 函数返回一个接近零的小值，用于防止分母为零的错误。

    def recall(y_true, y_pred):
  true_positives = K.sum(K.round(K.clip(y_true * y_pred, 0, 1)))
  possible_positives = K.sum(K.round(K.clip(y_true, 0, 1)))
  return true_positives / (possible_positives + K.epsilon())

    这段代码实现了一个召回率（recall）指标的计算函数，其中 y_true 表示真实标签，y_pred 表示模型预测的标签。该函数使用了Keras后端的函数和操作。

    具体解释如下：

    true_positives = K.sum(K.round(K.clip(y_true * y_pred, 0, 1)))：计算真正例数，即在真实标签和预测标签都为1的情况下，正确预测的样本数。首先将相乘结果截断到 [0,1] 区间内，然后四舍五入取整，最后计算总和。

    possible_positives = K.sum(K.round(K.clip(y_true, 0, 1)))：计算所有真实正例数，即所有真实标签为1的样本数。同样先将真实标签值截断到 [0,1] 区间，取整并计算总和。

    return true_positives / (possible_positives + K.epsilon())：最终返回召回率值，即真正例数除以所有真实正例数加上一个极小值 epsilon，用于稳定计算。

    需要注意的是，这里使用了Keras中的一些通用函数和操作，其中 K 是Keras的后端（backend）引擎，可以是TensorFlow、Theano等。epsilon() 函数返回一个接近零的小值，用于防止分母为零的错误。

    def f1(y_true, y_pred):
  p = precision(y_true, y_pred)
  r = recall(y_true, y_pred)
  return 2*((p*r)/(p+r))

    这段代码实现了一个常用的二分类模型性能评估指标F1-score。其中，y_true是真实标签向量，y_pred是预测标签向量。precision和recall分别计算了模型预测为正样本的正确率和召回率。

    具体而言，precision表示在模型预测为正例的样本中，有多少是真正的正例，其计算公式为：TP/(TP+FP)，其中TP为True Positive即真正例数量，FP为False Positive即假正例数量。

    recall表示在所有真实正例样本中，模型成功捕捉到了多少，其计算公式为：TP/(TP+FN)，其中TP为True Positive即真正例数量，FN为False Negative即假反例数量。

    F1-score是precision和recall的加权调和平均数，反映了模型的综合性能，计算公式为：2*(precision*recall)/(precision+recall)。

    因此，该函数返回的值就是由预测结果计算得出的F1-score。

    def load_word_vec(word_index=None, embedding_dim=100):
    fname = './glove.twitter.27B/glove.twitter.27B.'+str(embedding_dim)+'d.txt' \
        if embedding_dim != 300 else './glove.42B.300d.txt'
    fin = open(fname, 'r', encoding='utf-8', newline='\n', errors='ignore')
    word_vec = {}
    for line in fin:
        tokens = line.rstrip().split()
        if tokens[0] in word_index.keys():
            word_vec[tokens[0]] = np.asarray(tokens[1:], dtype='float32')
    return word_vec
