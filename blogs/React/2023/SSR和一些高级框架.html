    SSR和一些高级框架

    御坂10043 2023/3/6 React

    前端服务框架总结：

    # 一、Next.js

    React前端应用开发框架。 重心在 Web 部分，对 UI 呈现部分的代码的组织方式、服务器端渲染功能等提供了完善的支持。

    # 页面路由

    文件即路由，默认以pages文件夹为入口，生成对应的路由结构。 pages文件夹内所有文件都会被当作路由入口文件，支持多层级，会根据层级生成路由地址。index文件名会被忽略:/pages/user/index.js对应的访问地址是user。

    [api路由:API路由通常是指在客户端与服务器之间进行交互时使用的一组规则或约定。例如定义不同HTTP方法（如GET、POST等）应该执行什么操作，以及针对不同URL路径应该调用哪些服务器端函数等方面的规则。]

    # api路由

    9.x版本后支持api路由，在pages/api/文件夹下的文件会作为api生效，而不会进入react前端路由中，命名规则相同，pages/api/article/[id],js => /api/article/123。

    # 动态路由

    使用中括号命名：/pages/article/[id].js => pages/article/123

    # 路由加载

    提供link型组件。 当使用这个组件替代 标签进行路由跳转时，组件会检测链接是否命中路由，如果命中，则组件出现在视口后会触发对对应路由的 js 等资源的加载，并且点击跳转时使用路由跳转，不会重新加载页面，也不需要再等待获取渲染所需 js 等资源文件。

    # 出错兜底

    提供了错误码响应的兜底跳转。

    pages 文件夹下定义 http 错误码命名的页面路由。

    当其他路由发生响应错误时，就会跳转到到错误码路由页面。

    # 二、Nuxt.js

    Vue前端应用开发框架。 重心在 Web 部分，对 UI 呈现部分的代码的组织方式、服务器端渲染功能等提供了完善的支持。

    # 页面路由

    与Next相同，只是vue有官方自己的路由实现方式vue-router，因此在编译时会生成vue-router结构的路由配置。

    与某个路由文件同名的文件夹下的文件会变成子路由，如:
